Intro

There are three parts of a filter condition: filter parameter, filter comparator and the value.
Conditions can be combined with | (OR) and & (AND) to perform multi condition filtering.

  • AND (&) have a higher precedence than OR (|), meaning that the conditions combined with AND (&) will be considered first.
  • Parentheses currently not allowed.

All expressions are case-sensitive.

Valid Values

Filter Parameters

Parameter Expression Description Valid Values Example
State %state% Name or abbreviation of the state. (as described) WI or Wisconsin
County %county% Name of the county. When using this, state abbreviation needs to be attached. (as described) Dane, WI
Confirmed Cases %confirmed% Confirmed cases count. Positive Integer 57
Fatal Cases %fatal% Fatal cases count. Positive Integer 57
Confirmed Cases Per 100K Residents %confirmed_100k% Confirmed cases count per 100K residents. Positive Float Number which is ≤ 100K 157.8
Fatal Cases Per 100K Residents %fatal_100k% Fatal cases count per 100K residents. Positive Float Number which is ≤ 100K 157.8
Death Rate (%) %death_rate% Death Rate. 0.0 ≤ x ≤ 100.0 57.8
Latitude %lat% Latitude of the location. -90.0 ≤ x ≤ 90.0 57.8
Longitude %lon% Longitude of the location. -180.0 ≤ x ≤ 180.0 57.8
ZIP Code %zip% ZIP code of the location. 5-digit number 53716
Date %date% Date of the data. YYYY-MM-DD 2020-04-21

Filter Comparator

  • >= ‐ Greater than or equals
  • > ‐ Greater than
  • <= ‐ Less than or equals
  • < ‐ Less than or equals
  • = ‐ Equals

Examples

  • %confirmed% > 70
    Confirmed case is greater than 70.
  • %fatal% > 100 & %fatal_100k% > 3.1
    Fatal case is greater than 100 and fatal case per 100K residents is greater than 3.1.
  • %state% = WI | %state% = NY
    Filter the data to be the data of WI (Wisconsin) only or NY (New York).
  • %state% = WI & %confirmed% > 300 | %state% = IN & %confirmed% > 400
    Get the data in Wisconsin with the confirmed case count over 300 or the data in Indiana with the confirmed case count over 400.
